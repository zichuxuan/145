# 流程画布全屏与缩放实现计划

## Summary

* 目标：为“流程配置 -> 流程设计 -> 远程控制”画布增加“窗口级全屏画布”能力，并提供“放大、缩小、重置 100%”三个缩放按钮。

* 成功标准：

  * 在流程设计页进入远程控制后，可以一键切换到全屏画布态。

  * 全屏态下保留当前工作流上下文与节点编辑能力，不丢失已编辑内容。

  * 支持放大、缩小、重置到 100%，缩放后画布内容可继续滚动查看。

  * 退出全屏后返回原流程设计页，画布数据与当前缩放状态按约定保持一致。

## Current State Analysis

* 主窗口 `hmi/views/main_window.py` 已通过 `showFullScreen()` 全屏展示，因此这里的“全屏”不是应用窗口再全屏，而是流程设计内部的画布编辑区域进入独占态。

* `hmi/views/smart_production.py` 中的 `WorkflowCanvasEditor` 目前是普通 `QWidget`：

  * 内部由 `QScrollArea` + `canvas_widget` + `canvas_layout` 组成。

  * 当前只负责渲染开始/结束、连接器、普通节点、判断节点、循环节点。

  * 还没有工具栏、缩放状态、全屏切换、比例显示等能力。

* 流程设计页 `_create_workflow_designer_page()` 当前结构为：

  * 顶部返回/标题/存草稿/保存按钮。

  * 中部基础属性/远程控制 Tab。

  * 下方 `workflow_editor_stack`，其中远程控制页只直接放入 `WorkflowCanvasEditor`。

* 仓库内未发现现成的全屏画布、缩放按钮或缩放状态管理逻辑，可直接在 `smart_production.py` 内补齐第一版。

## Assumptions & Decisions

* 已确认的产品决策：

  * 全屏方式：窗口级全屏画布。

  * 缩放交互：第一版只做按钮缩放，包含“放大、缩小、重置 100%”。

* 具体实现决策：

  * 不改变主应用窗口的全屏状态；“窗口级全屏画布”落地为 `SmartProduction` 内部的工作流设计页切换到专用全屏画布视图。

  * 使用 `QStackedWidget` 在“普通流程设计页”和“全屏画布页”之间切换，避免引入跨窗口数据同步复杂度。

  * 缩放不改 `workflow_detail` 数据结构，只影响视图展示状态。

  * 第一版缩放精度采用离散步进，例如 `75% / 90% / 100% / 110% / 125% / 150%` 或固定步长 `0.1`；执行时统一封装，避免散落在 UI 代码里。

  * 退出全屏后保留当前缩放比例，除非用户点击“重置 100%”。

## Proposed Changes

### 1. `hmi/views/smart_production.py`

* 在 `WorkflowCanvasEditor` 中补充视图状态：

  * 增加缩放比例字段，如 `self.zoom_factor = 1.0`。

  * 增加缩放边界与步长常量，如最小 `0.5`、最大 `2.0`、步长 `0.1`。

  * 增加公开方法：`zoom_in()`、`zoom_out()`、`reset_zoom()`、`set_zoom_factor()`、`get_zoom_factor()`。

* 调整画布渲染容器，使缩放可作用于整棵画布内容：

  * 保持 `QScrollArea` 负责滚动。

  * 在 `canvas_widget` 外再增加一层内容容器，缩放时通过固定宽高 / 最小尺寸 / 样式比例策略驱动内容整体放大缩小。

  * 第一版不引入 `QGraphicsView`，继续沿用现有 QWidget 方案，降低对当前节点编辑实现的侵入。

* 在 `WorkflowCanvasEditor` 顶部新增工具栏区域：

  * 全屏按钮：进入/退出全屏画布态。

  * 缩小按钮。

  * 比例显示标签，如 `100%`。

  * 放大按钮。

  * 重置按钮。

* 增加全屏模式回调或信号：

  * 方案一：给 `WorkflowCanvasEditor` 注入回调。

  * 方案二：增加信号由 `SmartProduction` 处理。

  * 本次优先用 Qt signal，更符合当前 MVVM/UI 解耦风格。

* 在 `SmartProduction` 中重构远程控制页容器：

  * 现有 `_create_workflow_canvas_page()` 从“直接放一个编辑器”改成“普通画布页容器 + 全屏画布页容器”。

  * 新增一个内部 `QStackedWidget` 或等价页面切换结构，管理普通态和全屏态。

  * 两个页面共享同一个 `WorkflowCanvasEditor` 实例，或在切换时转移同一个编辑器，避免双实例导致状态分叉。

* 新增进入/退出全屏的方法：

  * `SmartProduction._enter_workflow_canvas_fullscreen()`

  * `SmartProduction._exit_workflow_canvas_fullscreen()`

  * 进入时隐藏流程设计页非必要区域，仅保留全屏画布。

  * 退出时恢复原有布局与按钮状态。

* 样式与交互细节：

  * 全屏画布态背景延续当前深色主题。

  * 全屏页顶部保留“返回流程设计”或“退出全屏”按钮。

  * 保存/存草稿不放在画布工具栏，避免和视图控制混淆，仍沿用设计页顶部按钮或在全屏头部只提供返回。

### 2. 交互与数据流约束

* 节点新增、编辑、删除仍通过当前 `WorkflowCanvasEditor` 内部逻辑执行，不新增后端交互。

* 全屏切换前后不触发重新加载 workflow API，不重置 `workflow_detail`。

* 缩放仅改变视觉，不改变节点配置、不改变保存 payload。

* 如果当前在全屏态执行节点编辑弹窗，弹窗仍以当前页面为父级弹出。

## Edge Cases

* 在最小缩放比例继续点击“缩小”：按钮置灰或保持比例不变。

* 在最大缩放比例继续点击“放大”：按钮置灰或保持比例不变。

* 画布内容超出可视范围：依旧依赖 `QScrollArea` 滚动查看。

* 判断/循环节点分支较宽时：缩小后仍可完整浏览；放大后仍允许横向滚动。

* 编辑已有工作流后进入全屏：必须沿用当前已回填的 `workflow_detail`，不得触发重置。

* 从全屏退出到列表页或切换 Tab：需要明确退出全屏，避免遗留在不可见页面中。

## Verification

* 静态验证：

  * 检查 `smart_production.py` 无语法错误与诊断错误。

  * 确认新增方法命名、信号连接、页面切换逻辑无死链。

* 功能验证：

  * 新建流程，切到远程控制，点击全屏按钮，确认画布进入窗口级独占态。

  * 在全屏态执行“放大、缩小、重置 100%”，确认比例显示正确变化。

  * 在不同缩放级别下新增节点、编辑节点、删除节点，确认交互仍可用。

  * 退出全屏后返回流程设计页，确认画布内容未丢失。

  * 编辑已有流程进入全屏，确认回显内容一致。

  * 执行一次存草稿与保存，确认 payload 不受缩放状态影响。
