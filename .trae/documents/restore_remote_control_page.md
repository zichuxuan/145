# 远程控制页面 Figma 还原计划

本计划旨在重构 `DeviceDialog` 类，以支持“基础属性”和“远程控制”两个标签页的切换，并完全按照 Figma 设计稿还原“远程控制”页面的 UI 样式和布局。

## 当前状态分析
- `DeviceDialog` 目前仅实现了“基础属性”页面的 UI。
- 界面中已有“基础属性”和“远程控制”两个按钮，但尚未实现切换逻辑。
- “远程控制”页面内容完全缺失。

## 提议方案

### 1. 结构重构 (Refactor)
- 在 `DeviceDialog` 中引入 `QStackedWidget` 作为主内容区域容器。
- 将现有的“基础属性”网格布局及其控件移动到一个独立的页面（QWidget）中。
- 为“远程控制”创建一个新的页面（QWidget）。
- 实现标签页按钮（`base_tab`, `remote_tab`）的点击事件，切换 `QStackedWidget` 的当前索引。

### 2. 样式更新 (Styles)
- 更新标签页按钮的样式，使其在选中和未选中状态下具有不同的视觉反馈（如 Figma 所示，选中时为蓝色背景，未选中时为透明/灰色）。
- 统一定义输入框、按钮和表格的样式常量，确保与 Figma 颜色值（如背景 `#1C212B`, 输入框 `#2A313E` 等）一致。

### 3. 实现“远程控制”页面 (Remote Control Page)
- **控制模式**: 使用 `QRadioButton` 实现“PLC控制”和“手动控制”，并应用自定义样式（圆点指示器）。
- **网络设置**: 实现“IP地址”和“端口”输入框，保持与“基础属性”页一致的高度和样式。
- **行为事件列表**:
    - 添加“行为事件”标题和“新增设备”按钮（位于表格上方）。
    - 实现一个自定义表格（基于 `QTableWidget` 或自定义布局），包含列：序号、事件名称、点位地址、功能码、偏移量、数据、描述、操作。
    - 表格行应用交替背景或特定行高/间距。
    - “操作”列添加“删除”按钮，样式为红框红字。
- **分页组件**: 在表格下方实现分页 UI（1, 2, 3, 4, 5 及翻页箭头）。

### 4. 数据交互 (Data Flow)
- 确保 `_populate_from_device` 和 `_on_confirm` 方法能够正确处理和保存“远程控制”页面的数据。

## 具体修改文件
- [dialogs.py](file:///home/pi/project/hmi/views/components/dialogs.py):
    - 修改 `DeviceDialog` 类，添加 `QStackedWidget`。
    - 重构 `_init_ui` 方法。
    - 新增 `_init_base_tab_ui` 和 `_init_remote_tab_ui` 方法。
    - 添加切换标签页的逻辑。

## 验证步骤
- 启动 HMI 应用并打开“设备编辑”弹窗。
- 点击“远程控制”标签页，检查页面是否正确切换。
- 对比 Figma 设计稿，检查：
    - 颜色值是否准确。
    - 间距（Margins/Spacing）是否一致。
    - 字段名称和样式是否完全匹配。
    - 表格和分页 UI 是否显示正常。
