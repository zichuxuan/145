# 判断节点配置弹窗布局重构设计

## 背景
当前工作流编辑器中，判断节点的配置弹窗布局较为冗余，包含多个用户不常修改的文案字段，且条件分支的布局不够直观。用户期望按照新的 UI 设计图重构该弹窗的布局和字段。

## 目标与原则
1. **统一风格**: 保持系统现有的深色主题（Dark Theme），仅调整布局和字段分布。
2. **精简字段**: 隐藏 `condition_name`、`yes_label` 和 `no_label` 字段，后台保存时自动使用默认值。
3. **优化布局**: 将条件分支分为「如果」和「否则」两个带边框的区块（Card）。
4. **条件行样式**: 第一行无删除按钮，后续行带有红色删除图标（🗑️）。逻辑连接符（且/或）独立占据一行，位于条件行之间。

## 详细设计

### 1. 标题与标签
- 移除原本的“配置判断”字样，改为居中的大号标题「判断」。
- 增加左对齐的小节标题「条件分支」。

### 2. “如果”条件分支卡片 (Yes Branch Card)
- **卡片容器**: 使用带有边框和微透明背景的 `QFrame`。
- **头部行**: 
  - 左侧: `QLabel("如果")`
  - 右侧: `QPushButton("增加条件")`，样式为浅蓝色文字、蓝色边框，背景透明。
- **条件列表区**:
  - 条件行: 水平排列 `[选择属性(2)]` `[判断符(2)]` `[请输入(3)]`。
  - 删除按钮: 第一行不显示，第二行及以后显示为红色垃圾桶图标，固定宽度。
  - 逻辑连接符: 下拉框（且/或）独立一行，左对齐，插入在两行条件之间（仅在当前行不是最后一行时显示）。

### 3. “否则”分支卡片 (No Branch Card)
- 独立的 `QFrame`，与“如果”卡片同宽。
- 内部仅包含一个左对齐的 `QLabel("否则")`，起到占位和提示作用。

### 4. 底部操作区
- 保留现有的「取消」和「确认」按钮，但居中显示。
- “确认”按钮为主按钮样式（蓝色背景）。

## 数据保存逻辑 (`get_config`)
由于移除了三个输入框，`get_config` 方法需要直接写入默认值：
```python
result["condition_name"] = "判断节点"
result["yes_label"] = "当满足时"
result["no_label"] = "否则"
```

## 测试验证
- 打开判断节点配置弹窗，确认布局是否与设计一致。
- 增加多个条件，确认逻辑下拉框和删除按钮的显示/隐藏状态。
- 保存节点并重新打开，确认条件数据是否正确回显。
- 确认保存到 `workflow_detail` 的 payload 包含正确的默认文案字段。
